## 目的 / In-Out / Related
- **目的**: ユーザーが自分の申請や承認待ち申請を確認する
- **対象範囲（In/Out）**: 一覧表示・フィルタ・ソート。申請詳細は SCR-B03
- **Related**: REQ-B03 / SPEC-API-B01 / DD-DB-workflows

## 画面情報
- **画面ID**: SPEC-SCR-B01
- **画面名**: 申請一覧
- **対象ロール**: 全ロール（表示内容がロールにより異なる）
- **状態**: Draft

### ページ構成

実装では「自分の申請」と「承認待ち」を**別ページ**として分離している。

| ページ | URL | フィルタ条件 | 対象ロール |
|---|---|---|---|
| 自分の申請一覧 | `/workflows` | `created_by = user.id` | 全ロール |
| 承認待ち一覧 | `/workflows/pending` | `approver_id = user.id AND status = 'submitted'`（`tenant_admin` は全件） | Approver / Tenant Admin |

- ページ間の遷移はサイドバーから行う
- 承認待ち一覧（`/workflows/pending`）は Approver / Tenant Admin 以外にはサイドバーに表示されない

## 主要ユースケース
- Member: 自分の申請一覧を確認し、ステータスを追跡する
- Approver: 承認待ちの申請を確認し、詳細画面へ遷移する

## UI構成（概要）
- **テーブル列**: 申請番号 / 種別 / タイトル / ステータス / 申請日 / 申請者
- **フィルタ**: ステータス（下書き/申請中/承認済/差戻し）、期間、種別
- **ソート**: 申請日（デフォルト降順）
- **ページネーション**: 20件/ページ

## 振る舞い・遷移
- 正常系: 行クリック → `/workflows/[id]`（SCR-B03）へ遷移
- 「新規申請」ボタン → `/workflows/new`（SCR-B02）へ遷移
- 異常系: データ取得失敗 → エラートースト表示

## エラー/例外
- データ0件: 「申請はありません」の空状態表示
- ネットワークエラー: リトライボタン付きエラー表示

## 監査ログポイント
- 一覧表示自体はログ対象外（閲覧のみ）
